ModuloHome novo

É a tela principal do módulo — a primeira coisa que o Jasap instancia quando o usuário clica em "MÓDULO" no menu do sistema. Ela não exibe conteúdo por si só: sua função é montar a estrutura da janela (layout, sidebar, header) e delegar o carregamento inicial para as inner classes.

É composta pela classe principal e quatro partes internas: MID, render(), Title, MenuItem e MenuInicial.

package br.xt.app.modulo;

import br.jasap.core.Effect;
import br.jasap.effect.Response;
import br.jasap.gui.JasapPage;
import br.jasap.gui.SideMenu;
import br.jasap.gui.Table;
import br.jasap.gui.TableRow;
import br.jasap.util.Js;
import br.xt.AppsRootAction;
import br.xt.acore.view.XtPage;

public class ModuloHome extends AppsRootAction {

    public static final String MID = "__MODULO_MID";

    @Override
    public Effect execute() throws Exception {
        render();
        return new Response();
    }

    public void render() throws Exception { ... }

    public static class Title extends ModuloHome { ... }
    public static class MenuItem extends ModuloHome { ... }
    public static class MenuInicial extends ModuloHome { ... }
}

Por que extends AppsRootAction?

AppsRootAction é a classe base de todas as actions do projeto. Ela fornece os métodos que usamos dentro do ModuloHome: ui(), link(), eval(), update(), getInput(), getFactory(), getManager() e outros. Sem ela, nada disso estaria disponível.

Por que as inner classes estendem ModuloHome?

Cada inner class (Title, MenuItem, MenuInicial) é uma action independente — o Jasap as registra e roteia separadamente. Ao estender ModuloHome, elas herdam todos os métodos de AppsRootAction e também têm acesso direto à constante MID e ao método render() sem precisar importar nada.

render()

public void render() throws Exception {
    XtPage page = new XtPage(getManager());
    if (isAjaxCall()) {
        eval(Js.CLOSE_SUB_WINDOWS);
    } else {
        page.getBody().append(page.content("", "", false));
        page.setWinTitle("Módulo");
        page.getCommands().add(link(MenuItem.class).putInteger(MID, 0).ajax());
        getOutput().write(this, page);
    }
}

O que faz

Monta e envia a página HTML completa do módulo ao browser. É chamado pelo execute() em toda requisição ao ModuloHome.

XtPage page = new XtPage(getManager())

Cria a estrutura da página. XtPage é a classe do projeto que monta o HTML completo — CSS, JS, layout do Jasap. Recebe o getManager() para saber o contexto do módulo.

getManager() retorna o ModuloManager injetado pelo framework em runtime. A ligação é feita quando o ModuloManager chama regAction(ModuloHome.class) — o Jasap armazena internamente esse mapeamento e injeta o manager antes de chamar o execute(). Por isso não há nenhum import de ModuloManager dentro do ModuloHome.

isAjaxCall() — por que dois caminhos?

O ModuloHome pode ser chamado em dois contextos diferentes:

ContextoComo chegaO que fazer
Primeira aberturaUsuário clica em "MÓDULO" no menu do sistemaMontar a página HTML inteira e enviar ao browser
AjaxUsuário já está no módulo e navega de volta ao homeApenas fechar os modais abertos — a página já existe

Sem essa distinção, toda navegação interna recarregaria a página inteira, perdendo o estado da tela.

page.content("", "", false)

Monta o layout com três áreas: sidebar (#sidebar), header (navbar) e área de conteúdo (DIV_WINDOW). Os dois primeiros parâmetros estão vazios porque o conteúdo real será carregado depois via ajax pelo MenuItem. O false indica que é uma janela filha — não a página raiz do desk.

page.getCommands().add(...)

Registra um JavaScript que executa automaticamente após a página terminar de carregar no browser. Ele faz uma requisição ajax para o MenuItem com MID=0, que por sua vez carrega o sidebar e o breadcrumb. Sem essa linha a tela abriria completamente vazia.

eval(Js.CLOSE_SUB_WINDOWS)

Executado apenas no caso ajax. Fecha todos os modais abertos dentro do módulo antes de voltar ao menu inicial — evita que formulários ou listas sobrepostos fiquem visíveis por cima do home.

Title

public static class Title extends ModuloHome {
    @Override
    public Effect execute() throws Exception {
        int mid = getInput().getInteger(MID, 0);

        Table aux = new Table(getManager());
        TableRow line = aux.row();

        if (mid == 0) {
            line.col().setStyle("padding: 0 5 0 5;")
                .setContent(ui().btTitleActive("MÓDULO", ""));
        } else {
            line.col().setStyle("padding: 0 5 0 5;")
                .setContent(ui().btTitle("MÓDULO", link(Title.class).putInteger(MID, 0).ajax()));
        }

        update(JasapPage.DIV_HEADER_LEFT, aux);
        update(JasapPage.DIV_HEADER_CENTER, "");
        return new Response();
    }
}

O que faz

Atualiza o breadcrumb exibido no header (navbar) da janela. É chamada via ajax pelo MenuItem sempre que o usuário navega entre seções do módulo.

Por que usa Table para montar o breadcrumb?

O Jasap monta o header com componentes de layout — não aceita HTML direto. Table e TableRow são componentes do framework que geram o HTML estruturado esperado pelo DIV_HEADER_LEFT. Cada col() vira uma célula no breadcrumb.

mid == 0 vs mid > 0

SituaçãoComportamentoPor quê
Na raiz do módulo (MID=0)"MÓDULO" aparece como texto ativo, sem linkNão faz sentido clicar em "MÓDULO" se já está no módulo
Dentro de uma entidade (MID>0)"MÓDULO" vira link clicável que volta ao MID=0Permite navegar de volta ao menu inicial sem fechar a janela

Conforme entidades forem adicionadas, um switch(mid) exibirá o nome da entidade ativa ao lado de "MÓDULO" no breadcrumb.

update(DIV_HEADER_LEFT, aux)

Substitui o conteúdo da área esquerda do header via ajax, sem recarregar a página. DIV_HEADER_CENTER é limpo para remover qualquer conteúdo de telas anteriores que possa ter ficado.

MenuItem

public static class MenuItem extends ModuloHome {
    @Override
    public Effect execute() throws Exception {
        int mid = getInput().getInteger(MID, 0);

        if (mid > 0) {
            eval(link(Title.class).putInteger(MID, mid).ajax());
            eval(link(Title.class).putInteger(MID, mid).ajax(false));
        } else {
            eval(link(MenuInicial.class).ajax(false));
            eval(link(Title.class).putInteger(MID, mid).ajax(false));
        }

        return new Response();
    }
}

O que faz

É o roteador central do módulo. Toda navegação interna passa por aqui — o MID recebido determina o que será carregado na tela.

Por que existe em vez de chamar as actions diretamente?

Centralizar a navegação no MenuItem garante que o breadcrumb (Title) seja sempre atualizado independentemente de qual entidade está sendo aberta. Sem ele, cada link da sidebar precisaria lembrar de atualizar o header — um acoplamento desnecessário.

MID = 0 — menu inicial

Chama MenuInicial (carrega o sidebar) e Title (atualiza o breadcrumb). Os dois são disparados como requisições ajax encadeadas via eval().

MID > 0 — dentro de uma entidade

Atualiza o breadcrumb e abre a lista da entidade correspondente. Conforme entidades forem criadas, um switch(mid) será adicionado aqui:

switch (mid) {
    case 1: eval(preAjax(ModuloXxxList.class).modalMax()); break;
    case 2: eval(preAjax(ModuloYyyList.class).modalMax()); break;
}

ajax() vs ajax(false)

ChamadaComportamento
eval(link(...).ajax())Dispara a requisição imediatamente
eval(link(...).ajax(false))Enfileira — executada após as anteriores terminarem

O encadeamento garante que o sidebar ou a lista seja carregada depois que o breadcrumb já foi atualizado.

MenuInicial

public static class MenuInicial extends ModuloHome {
    @Override
    public Effect execute() throws Exception {
        SideMenu m = new SideMenu();

        update("sidebar", m.toHtml());
        return new Response();
    }
}

O que faz

Monta o HTML do menu lateral (sidebar) e envia ao browser via ajax. É chamada pelo MenuItem quando MID=0.

Por que é uma action separada e não está dentro do render()?

O sidebar é carregado via ajax depois que a página já está no browser. Se fosse montado dentro do render(), precisaria existir antes da página carregar — isso tornaria impossível atualizar só o sidebar sem recarregar a tela inteira. Sendo uma action separada, o MenuItem pode chamá-la a qualquer momento para reconstruir o sidebar.

Estado atual

m.add(true, "NomeDaEntidade", link(MenuItem.class).putInteger(MID, 1).ajax(), null)
 .setImage("res/ico/mn_cto_clientes.png");

update("sidebar", m.toHtml())

Substitui o conteúdo do elemento com id="sidebar" na tela sem recarregar a página. m.toHtml() serializa o SideMenu para HTML. O id "sidebar" é o mesmo que o Jasap usa na estrutura montada por page.content() — é por isso que o update funciona.
