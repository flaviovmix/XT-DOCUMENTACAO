AppsRootModelFactory editar arquivo

0:00 / 0:00

Centraliza o acesso às factories de todos os módulos do sistema. É o objeto retornado por getFactory() em qualquer action — o ponto único de entrada para acessar os models de qualquer módulo a partir de qualquer outro.

package br.xt;

import br.jasap.core.AppManager;
import br.xt.app.contatos.ContatosMdFactory;
import br.xt.app.ged.GedMdFactory;
import br.xt.app.laboratorio.LaboratorioMdFactory;
import br.xt.app.modulo.ModuloMdFactory;
import br.xt.app.painel.PainelMdFactory;

public class AppsRootModelFactory {

    protected AppManager manager = null;

    public AppManager getManager() {
        return manager;
    }

    public void setManager(AppManager manager) {
        this.manager = manager;
    }

    public AppsRootModelFactory() {
    }

    public AppsRootModelFactory(AppManager manager) {
        setManager(manager);
    }

    public ContatosMdFactory contatos() { return new ContatosMdFactory(getManager()); }

    public PainelMdFactory painel() { return new PainelMdFactory(getManager()); }

    public GedMdFactory ged() { return new GedMdFactory(getManager()); }

    public LaboratorioMdFactory laboratorio() { return new LaboratorioMdFactory(getManager()); }

    public ModuloMdFactory modulo() { return new ModuloMdFactory(getManager()); }

}

getFactory() e AppsRootModelFactory

Toda action do sistema herda de AppsRootAction, que expõe o método getFactory(). Esse método retorna uma instância de AppsRootModelFactory já com o manager da requisição atual. O resultado é que qualquer action pode acessar qualquer model do sistema com uma cadeia simples:

getFactory().laboratorio().pesModel()   // model de Pessoa do Laboratório
getFactory().modulo().xxxModel()        // model de Xxx do Módulo
getFactory().painel().usuarioModel()    // model de Usuário do Painel

Sem esse arquivo, getFactory() não saberia da existência do módulo — e nenhuma action conseguiria acessar seus models.

public ModuloMdFactory modulo()

public ModuloMdFactory modulo() { return new ModuloMdFactory(getManager()); }

Linha adicionada ao criar o módulo. Cada chamada a getFactory().modulo() cria uma nova instância de ModuloMdFactory repassando o manager da requisição. A partir daí, os models do módulo ficam acessíveis via getFactory().modulo().pesModel() etc.

ElementoO que significa
publicPrecisa ser público — é chamado de dentro de qualquer action do sistema
ModuloMdFactoryTipo retornado. O Java usa esse tipo para saber quais métodos estão disponíveis após o getFactory().modulo()
modulo()Convenção: nome do módulo em minúsculas. Segue o padrão dos outros — laboratorio(), painel(), ged()
new ModuloMdFactory(getManager())Cria e retorna uma nova instância da factory, passando o manager da requisição atual. A factory usa esse manager para repassar aos models que criar

Diferença em relação à ModuloMdFactory: aqui não há lazy initialization — cada chamada a modulo() cria uma nova instância. O lazy init acontece dentro da própria ModuloMdFactory, nos métodos pesModel(), prfModel() etc., que garantem uma única instância por factory. Como a factory é criada por chamada, o efeito prático é o mesmo: uma instância de cada model por requisição.
