PnlCfg editar arquivo

0:00 / 0:00

Tela de configurações gerais do sistema. É aqui que o administrador habilita ou desabilita cada módulo. Ao criar um novo módulo, é preciso registrá-lo neste arquivo em quatro lugares — sem isso, o método PnlCfg.hasModulo() não existe e o HomeDesk não compila.

public class PnlCfg extends AppsRootAction {

    public static class Insert extends PnlCfg {
        public Insert() { super(); super.getFilters().add(new TransactionFilter()); }
        @Override
        public Effect execute() throws Exception {
            // ...
            // ADICIONAR ↓
            getFactory().painel().cfgModel().insert(MD_MODULO, getInput().getString(MD_MODULO));
            getFactory().painel().cfgModel().insert(MD_LABORATORIO, getInput().getString(MD_LABORATORIO));

            eval(Js.alert("Informações salvas com sucesso!"));
            return new Response();
        }
    }

    // ...

    public String form() throws Exception {
        form = ui().form();
        // ...
        if (isAdminXT()){
            // ...
            // ADICIONAR ↓
            form.line().add(mdModulo(), "200");
            form.line().add(mdLaboratorio(), "200");
        }
        return form.getTable().toHtml();
    }

    // ...

    // ADICIONAR — método novo ↓
    protected Check mdModulo = null;
    public Check mdModulo() throws Exception {
        JasapList list = new JasapList();
        list.getList().add(new DomainValue(1, "ativar Módulo MÓDULO"));

        if (mdModulo==null){
            mdModulo = new Check(getManager(), MD_MODULO)
            .setLabel("")
            .setValue(list);
            mdModulo.getValue().setSelected(getFactory().painel().cfgModel().getSetting(MD_MODULO));
        }
        return mdModulo;
    }
    // FIM mdModulo() ↑

    // ADICIONAR — método novo ↓
    public static boolean hasModulo(AppManager manager) throws Exception{
        PcfgModel cfgModel = new PcfgModel(manager);
        return (cfgModel.getSetting(MD_MODULO)!=null);
    }
    // FIM hasModulo() ↑

    // ...

    // ADICIONAR — constante nova ↓
    public static String MD_MODULO = ROOT.concat("__MD_MODULO");

    // ...
}

O que muda neste arquivo

São quatro adições independentes, todas seguindo o mesmo padrão dos outros módulos já registrados no arquivo.

O que adicionarOnde
getFactory().painel().cfgModel().insert(MD_MODULO, ...)Dentro de Insert.execute() — persiste o valor do checkbox no banco ao salvar
form.line().add(mdModulo(), "200")Dentro de form(), no bloco if (isAdminXT()) — exibe o checkbox na tela de configuração
mdModulo() + hasModulo()Métodos novos na classe — o primeiro monta o componente visual, o segundo é usado pelo HomeDesk
MD_MODULOConstante nova no final da classe — chave usada para identificar a configuração no banco

MD_MODULO

public static String MD_MODULO = ROOT.concat("__MD_MODULO");

Chave de configuração do módulo no banco de dados. É o identificador único que o sistema usa para ler e gravar o estado do checkbox — ativado ou desativado. O prefixo ROOT garante que a chave fique no namespace correto e não conflite com outras configurações.

ElementoO que significa
public staticVisível e acessível sem instanciar a classe — necessário porque Insert, mdModulo() e hasModulo() referenciam essa constante diretamente
ROOT.concat("__MD_MODULO")Concatena o prefixo ROOT com o identificador único do módulo. Resultado: "__ROOT_CFG_SYSTEM/__MD_MODULO"

mdModulo() e hasModulo()

protected Check mdModulo = null;
public Check mdModulo() throws Exception {
    JasapList list = new JasapList();
    list.getList().add(new DomainValue(1, "ativar Módulo MÓDULO"));

    if (mdModulo==null){
        mdModulo = new Check(getManager(), MD_MODULO)
        .setLabel("")
        .setValue(list);
        mdModulo.getValue().setSelected(getFactory().painel().cfgModel().getSetting(MD_MODULO));
    }
    return mdModulo;
}

public static boolean hasModulo(AppManager manager) throws Exception{
    PcfgModel cfgModel = new PcfgModel(manager);
    return (cfgModel.getSetting(MD_MODULO)!=null);
}

mdModulo() monta o componente visual do checkbox que aparece na tela de configuração. hasModulo() é o método estático usado pelo HomeDesk para decidir se o ícone do módulo aparece no desk — se o checkbox não estiver marcado, o módulo some da tela inicial para todos os usuários.

TrechoO que faz
new Check(getManager(), MD_MODULO)Cria um campo checkbox vinculado à chave MD_MODULO. O Jasap usa essa chave para enviar e receber o valor no formulário
new DomainValue(1, "ativar Módulo MÓDULO")Define o único item do checkbox — valor 1 quando marcado, e o texto que aparece ao lado
mdModulo.getValue().setSelected(...getSetting(MD_MODULO))Pré-marca o checkbox com o valor atual salvo no banco. Se o módulo já estava ativo, o checkbox já abre marcado
hasModulo(AppManager manager)Consulta o banco e retorna true se a chave MD_MODULO tiver algum valor salvo (checkbox marcado). Retorna false se o valor for null (nunca marcado ou desmarcado)
static em hasModuloPermite chamar PnlCfg.hasModulo(getManager()) de qualquer lugar sem instanciar PnlCfg. O HomeDesk usa exatamente assim

Insert

getFactory().painel().cfgModel().insert(MD_MODULO, getInput().getString(MD_MODULO));

Persiste o valor do checkbox no banco quando o administrador clica em "Salvar". O cfgModel().insert() grava o par chave-valor: se o checkbox estava marcado, salva o valor; se estava desmarcado, salva null. É esse null que o hasModulo() detecta para retornar false.

form()

if (isAdminXT()){
    // ...
    form.line().add(mdModulo(), "200");
    form.line().add(mdLaboratorio(), "200");
}

Adiciona o checkbox do módulo na tela de configuração. O bloco if (isAdminXT()) restringe a visibilidade — apenas administradores XT enxergam os módulos de estudo/desenvolvimento. A ordem das linhas dentro do bloco determina a ordem visual dos checkboxes na tela.
