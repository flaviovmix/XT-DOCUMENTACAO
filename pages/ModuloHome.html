ModuloHome novo arquivo

0:00 / 0:00

Tela inicial do módulo. Controla a navegação lateral (sidebar), o breadcrumb do header e o roteamento entre as entidades via parâmetro MID. É a primeira action instanciada quando o usuário abre o módulo.

package br.xt.app.modulo;

import br.jasap.core.Effect;
import br.jasap.effect.Response;
import br.jasap.gui.JasapPage;
import br.jasap.gui.SideMenu;
import br.jasap.gui.Table;
import br.jasap.gui.TableRow;
import br.jasap.util.Js;
import br.xt.AppsRootAction;
import br.xt.acore.view.XtPage;

public class ModuloHome extends AppsRootAction {

    public static final String MID = "__MOD_MID";

    @Override
    public Effect execute() throws Exception {
        render();
        return new Response();
    }

    public void render() throws Exception {
        XtPage page = new XtPage(getManager());
        if (isAjaxCall()) {
            eval(Js.CLOSE_SUB_WINDOWS);
        } else {
            page.getBody().append(page.content("", "", false));
            page.setWinTitle("Módulo");
            page.getCommands().add(link(MenuItem.class).putInteger(MID, 0).ajax());
            getOutput().write(this, page);
        }
    }

    public static class Title extends ModuloHome {
        @Override
        public Effect execute() throws Exception {
            int mid = getInput().getInteger(MID, 0);

            Table aux = new Table(getManager());
            TableRow line = aux.row();

            line.col().setStyle("padding: 0 5 0 5;").setContent(ui().btTitleActive("MÓDULO", ""));

            update(JasapPage.DIV_HEADER_LEFT, aux);
            update(JasapPage.DIV_HEADER_CENTER, "");
            return new Response();
        }
    }

    public static class MenuItem extends ModuloHome {
        @Override
        public Effect execute() throws Exception {
            int mid = getInput().getInteger(MID, 0);

            if (mid > 0) {
                eval(link(Title.class).putInteger(MID, mid).ajax());
                eval(link(Title.class).putInteger(MID, mid).ajax(false));
            } else {
                eval(link(MenuInicial.class).ajax(false));
                eval(link(Title.class).putInteger(MID, mid).ajax(false));
            }

            return new Response();
        }
    }

    public static class MenuInicial extends ModuloHome {
        @Override
        public Effect execute() throws Exception {
            SideMenu m = new SideMenu();

            update("sidebar", m.toHtml());
            return new Response();
        }
    }

}

ElementoO que significa
ModuloHomeNome da classe. Em Java, o nome identifica o arquivo e o papel que ele desempenha no sistema. Convenção do projeto: prefixo do módulo + Home. É a tela raiz — o ponto de entrada quando o usuário abre o módulo
extends AppsRootActionHerança em Java significa que esta classe recebe todos os recursos de AppsRootAction sem precisar reescrevê-los. É como herdar ferramentas de um pai: ui(), link(), eval(), update(), getInput(), getManager() e todos os métodos do framework ficam disponíveis automaticamente. Sem esse extends, nenhum desses métodos existiria nesta classe

public static final String MID = "__MOD_MID"

public static final String MID = "__MOD_MID";

Quando o usuário clica em um item do menu lateral, o sistema precisa saber o que foi clicado. Para isso, cada item do menu envia um número junto com a requisição — esse número é o MID. A constante MID guarda o nome da chave usada para transportar esse número entre o browser e o servidor.

ElementoO que significa
publicVisível para outras classes. Necessário porque outras actions precisam ler o valor de MID para montar links com esse parâmetro
staticPertence à classe, não a um objeto criado dela. Permite usar ModuloHome.MID diretamente, sem precisar instanciar a classe
finalO valor não pode ser alterado depois de definido. Garante que ninguém acidentalmente sobrescreva a chave durante a execução
MIDAbreviação de Menu ID. Identifica qual item do menu lateral foi clicado
"__MOD_MID"O texto que viaja na requisição HTTP como nome do parâmetro. O prefixo __ e o sufixo do módulo evitam conflito com parâmetros de outros módulos que possam existir na mesma sessão
Valor de MIDO que abre
0Menu inicial — exibe o sidebar. Conforme entidades forem adicionadas ao módulo, os itens aparecerão aqui
1, 2, 3…Cada entidade adicionada ao módulo receberá um MID. O switch(mid) em MenuItem decidirá qual tela abrir

Todo método execute() no Jasap é obrigatório — é ele que o framework chama quando uma requisição chega. Aqui ele apenas delega para render(). Essa separação existe por um motivo prático: as inner classes Title, MenuItem e MenuInicial às vezes precisam re-renderizar a home. Com render() separado, elas chamam esse método diretamente sem duplicar código.

O render() lida com dois cenários distintos:

SituaçãoO que acontece
Primeira abertura do móduloO browser ainda não tem a página. O servidor monta o HTML completo (estrutura, CSS, JS) e envia tudo de uma vez
Usuário já está no módulo e navega de volta ao homeA página já existe no browser. Não faz sentido reenviar tudo — só é necessário fechar janelas que estejam abertas por cima
TrechoO que faz
new XtPage(getManager())Cria o objeto que representa a página HTML. XtPage já inclui todos os CSS e JS do projeto. O getManager() informa qual módulo está sendo carregado — o Jasap usa isso para saber quais actions estão disponíveis
isAjaxCall()Verifica se a requisição veio de um clique dentro da aplicação (Ajax) ou de uma abertura direta no browser. Retorna true para Ajax
eval(Js.CLOSE_SUB_WINDOWS)Envia um comando JavaScript ao browser para fechar todos os modais abertos. Sem isso, o usuário voltaria ao home mas ainda veria formulários ou listas sobrepostos na tela
page.content("", "", false)Define a estrutura da página: cria o sidebar vazio (#sidebar), o header (navbar) e a área de conteúdo. Os parâmetros vazios indicam que o conteúdo será preenchido depois por Ajax — a página abre com a estrutura pronta mas sem dados
page.setWinTitle("Módulo")Define o texto que aparece na aba do navegador
page.getCommands().add(...)Registra um comando que será executado automaticamente assim que a página terminar de carregar no browser. Aqui dispara MenuItem com MID=0 — sem isso a página abriria com o sidebar completamente vazio
getOutput().write(this, page)Finaliza e envia a página como resposta HTTP ao browser. Só é chamado na abertura inicial — nas chamadas Ajax, o conteúdo é atualizado por partes via eval() e update()

Responsável por atualizar o breadcrumb no topo da tela — a "trilha de navegação" que indica onde o usuário está. É uma inner class: definida dentro de ModuloHome, mas tratada pelo Jasap como uma action independente com sua própria URL. Por estender ModuloHome, tem acesso direto à constante MID e a todos os métodos do framework.

No estado atual do módulo — sem entidades registradas — o breadcrumb exibe sempre "MÓDULO" como texto fixo, sem link. Quando entidades forem adicionadas, um if (mid == 0) será introduzido aqui: no menu inicial continuará com btTitleActive; dentro de uma entidade, "MÓDULO" virará link clicável via btTitle e o nome da entidade aparecerá ao lado.

TrechoO que faz
getInput().getInteger(MID, 0)Lê o número MID que veio na requisição. O segundo argumento (0) é o valor padrão caso o parâmetro não venha — evita NullPointerException. O valor é lido mas ainda não usado para lógica condicional; está presente para quando as entidades forem adicionadas
new Table(getManager())O Jasap não aceita HTML direto no header — exige componentes de layout. Table e TableRow geram a estrutura que o DIV_HEADER_LEFT espera receber
ui().btTitleActive("MÓDULO", "")Cria o botão de breadcrumb no estado ativo — texto sem link. O segundo parâmetro vazio confirma que não há ação ao clicar. É o estado fixo enquanto o módulo ainda não tem entidades
update(DIV_HEADER_LEFT, aux)Substitui o conteúdo do header esquerdo pelo breadcrumb montado. O Jasap atualiza apenas esse pedaço da tela — o resto continua intacto
update(DIV_HEADER_CENTER, "")Limpa o centro do header. Sem isso, conteúdo colocado ali por outra tela permaneceria visível após a navegação

MenuItem

public static class MenuItem extends ModuloHome {
    @Override
    public Effect execute() throws Exception {
        int mid = getInput().getInteger(MID, 0);

        if (mid > 0) {
            eval(link(Title.class).putInteger(MID, mid).ajax());
            eval(link(Title.class).putInteger(MID, mid).ajax(false));
        } else {
            eval(link(MenuInicial.class).ajax(false));
            eval(link(Title.class).putInteger(MID, mid).ajax(false));
        }

        return new Response();
    }
}

É o roteador central do módulo. Todo link do sidebar aponta para MenuItem com um MID diferente. A partir do valor recebido, MenuItem decide o que acontece: abre a tela da entidade ou volta ao menu inicial. Centralizar aqui garante que qualquer item do menu execute sempre a sequência correta — sem repetir lógica em cada link.

O comportamento é dividido em dois casos pelo if (mid > 0):

SituaçãoO que acontece
mid > 0Usuário clicou em uma entidade do menu. Quando entidades forem adicionadas, um eval(preAjax(XxxList.class).modalMax()) será inserido aqui para abrir a lista. Por enquanto, as duas chamadas a Title preparam o breadcrumb para receber o nome da entidade assim que ela existir
mid == 0Usuário voltou ao menu inicial. O sidebar é reconstruído via MenuInicial e o breadcrumb é redefinido via Title
TrechoO que faz
getInput().getInteger(MID, 0)Lê o número enviado pelo link clicado. Se nenhum MID vier, usa 0 como padrão — garante que a navegação sempre tenha um valor válido
eval(link(Title.class)...ajax())Chama Title imediatamente para atualizar o breadcrumb no mesmo ciclo da requisição
eval(link(Title.class)...ajax(false))Agenda uma segunda chamada a Title para após o ciclo atual — garante que o breadcrumb seja atualizado mesmo quando outras ações acontecem em sequência. Quando entidades forem adicionadas, esta linha permanecerá aqui enquanto o preAjax da lista será inserido acima dela
eval(link(MenuInicial.class).ajax(false))Reconstrói o sidebar via MenuInicial. O ajax(false) enfileira a execução para depois do ciclo atual

MenuInicial

public static class MenuInicial extends ModuloHome {
    @Override
    public Effect execute() throws Exception {
        SideMenu m = new SideMenu();
        update("sidebar", m.toHtml());
        return new Response();
    }
}

O sidebar não é montado junto com a página — ele é carregado separadamente via Ajax, após a página já estar no browser. Isso é necessário porque o sidebar precisa ser atualizado de forma independente: quando o usuário volta ao menu inicial, só o sidebar muda, o restante da tela continua intacto.

Se o sidebar fosse montado dentro do render(), seria impossível atualizá-lo sem recarregar a página inteira.

No estado atual do módulo o SideMenu é criado vazio — ainda não há entidades. Conforme entidades forem adicionadas, cada uma ganhará um m.add(...) aqui.

TrechoO que faz
new SideMenu()Cria o componente de menu lateral do Jasap. Internamente gera o HTML da sidebar com estilos e comportamento já definidos pelo framework — não é necessário escrever HTML manualmente
m.add(visivel, label, link, icone)Quando entidades forem adicionadas: o primeiro parâmetro controla se o item aparece (true/false), o label é o nome exibido, e o link aponta para MenuItem com o MID da entidade
m.toHtml()Converte o objeto SideMenu em HTML puro para ser enviado ao browser
update("sidebar", m.toHtml())Substitui o conteúdo do elemento com id="sidebar" na tela pelo HTML gerado. O Jasap injeta o novo conteúdo diretamente no DOM, sem recarregar a página
