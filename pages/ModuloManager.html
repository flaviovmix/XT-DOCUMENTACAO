ModuloManager novo arquivo

0:00 / 0:00

Responsável por registrar todas as actions do módulo no framework e definir a chave de permissão de acesso.

package br.xt.app.modulo;

import br.xt.app.painel.PnlManager;

public class ModuloManager extends PnlManager {

    public static String F_ACESSO_MODULO = "XT.PAINEL_CONTROLE.ACESSO_MODULO.MODULO";

    @Override
    public void config() throws Exception {

        regFun("PAINEL DE CONTROLE", "Acesso ao Módulo", "MÓDULO", F_ACESSO_MODULO);

        regAction(ModuloHome.class);
        regAction(ModuloHome.Title.class);
        regAction(ModuloHome.MenuItem.class);
        regAction(ModuloHome.MenuInicial.class);

    }
}

public class ModuloManager extends PnlManager

ElementoO que significa
publicA classe é visível para todo o projeto — outros pacotes como RootManager precisam instanciá-la, então não pode ser privada ou de pacote
classDeclara que isso é uma classe Java — o bloco que agrupa os dados e comportamentos do manager
ModuloManagerNome da classe. Convenção do projeto: prefixo do módulo + Manager
extends PnlManagerHerda de PnlManager, que fornece os métodos regAction() e regFun(). Sem essa herança não haveria como registrar actions nem permissões

public static String F_ACESSO_MODULO = "XT.PAINEL_CONTROLE.ACESSO_MODULO.MODULO"

ElementoO que significa
publicVisível fora da classe — necessário porque HomeDesk e outros arquivos leem essa constante diretamente via ModuloManager.F_ACESSO_MODULO
staticPertence à classe, não a uma instância. Permite usar ModuloManager.F_ACESSO_MODULO sem precisar criar um new ModuloManager()
StringTipo do valor — é uma cadeia de texto, não um número ou objeto
F_ACESSO_MODULONome da constante. Convenção do projeto: prefixo F_ indica funcionalidade de permissão. Maiúsculas porque é uma constante
"XT.PAINEL_CONTROLE.ACESSO_MODULO.MODULO"Chave única de permissão no sistema. O formato de pontos é uma hierarquia: XT (sistema) → PAINEL_CONTROLE (área) → ACESSO_MODULO (tipo) → MODULO (este módulo). Deve ser único no sistema inteiro — dois módulos com a mesma chave compartilhariam a permissão

@Override

Informa ao compilador que este método existe na classe pai (PnlManager) e está sendo substituído aqui. Se o nome do método estiver errado, o compilador avisa. Não é obrigatório, mas é boa prática.

public void config() throws Exception

ElementoO que significa
publicO RootManager chama new ModuloManager().config() de fora da classe — precisa ser público
voidO método não retorna nenhum valor — só executa os registros
config()Nome definido pela interface do Jasap. O framework chama esse método para inicializar o manager
throws ExceptionDeclara que o método pode lançar exceções. O Jasap exige isso porque os métodos internos de registro podem falhar

regFun("PAINEL DE CONTROLE", "Acesso ao Módulo", "MÓDULO", F_ACESSO_MODULO)

Registra a permissão no sistema de controle de acesso. Cria a entrada visível no Painel de Controle onde o admin pode liberar ou bloquear o acesso por usuário.

ParâmetroO que é
"PAINEL DE CONTROLE"Grupo onde a permissão aparece no Painel de Controle
"Acesso ao Módulo"Sub-grupo dentro do grupo
"MÓDULO"Nome exibido para o admin ao configurar permissões
F_ACESSO_MODULOA chave única que identifica essa permissão no sistema

regAction(ModuloHome.class)

Registra cada action no Jasap. Sem esse registro, o framework não conhece a action e não roteia nenhuma requisição para ela — é como se a classe não existisse para o sistema.

ParteO que significa
regActionMétodo herdado de PnlManager que adiciona a action ao mapa interno do Jasap
ModuloHome.classReferência à classe Java em tempo de compilação. O .class retorna o objeto Class<?> que representa a classe — o Jasap usa isso para instanciar a action quando uma requisição chegar

Cada inner class de ModuloHome também precisa ser registrada separadamente — Title, MenuItem e MenuInicial são actions independentes com suas próprias URLs no framework.
