ModuloProdutoList novo arquivo

Capítulo 1 — Visão geral ▼

Tela de listagem de Produto. Exibe os registros em uma ListView paginável com ordenação por coluna, botão para criar novo registro e exclusão inline com confirmação. Ao clicar numa linha, o formulário de edição é aberto em modal.

O arquivo concentra tudo relacionado à listagem num único lugar: a estrutura visual (render, window, br), o componente central (lView) e as actions de suporte (Sort, DeleteFromList). Se precisar mexer na lista, você abre um arquivo só.

Capítulo 2 — execute() e Sort ▼ 
@Override
public Effect execute() throws Exception {
    render();
    return new Response();
}

public static class Sort extends ModuloProdutoList {
    @Override
    public Effect execute() throws Exception {
        update(lView().getDIV_HEADER(), lView().getHeader());
        update(lView().getDIV_BODY(),   lView().getBody());
        update(lView().getDIV_NAVIGATE(), lView().getNavForm());
        return new Response();
    }
}

execute() é o ponto de entrada — apenas chama render() e retorna. Toda a lógica de montagem fica distribuída nos métodos da própria classe.

Sort é chamada quando o usuário clica num cabeçalho de coluna para ordenar ou navega entre páginas. Em vez de chamar render() de novo — o que redesenharia a tela inteira — ela atualiza apenas as três partes que realmente mudaram:

ParteO que contém
getDIV_HEADER()Cabeçalho das colunas — a seta de ordenação muda de coluna
getDIV_BODY()Linhas de dados — a página ou a ordem mudou
getDIV_NAVIGATE()Navegação de páginas — o número da página atual mudou

O resultado é o mesmo que redesenhar tudo, mas sem piscar e mais rápido — só o que mudou é substituído no browser via Ajax.

Capítulo 3 — render() e window() ▼ 
public void render() throws Exception {
    XtPage page = new XtPage(getManager());
    if (isAjaxCall()) {
        update(JasapPage.DIV_WINDOW, page.content(window().toHtml()));
    } else {
        page.getTable()
                .setBorder(4)
                .rowC("100%")
                .setContent(page.content(window().toHtml()))
                .setStyle(ui().stretchBorder());
        page.setWinTitle("Produtos");
        getOutput().write(this, page);
    }
}

public Table window() throws Exception {
    Table w = new Table(getManager()).setSize("100%", "100%");
    w.rowC("99%", JasapPage.DIV_WSPACE, lView());
    w.rowC("1%",  null, ui().line());
    w.rowC("1%",  JasapPage.DIV_BOTTOM, br());
    w.rowC("1%",  null, ui().line());
    return w;
}

render() decide como entregar a tela ao browser. O método isAjaxCall() distingue os dois casos:

ContextoO que acontece
Chamada Ajax (ex: abertura via modal)update(DIV_WINDOW, ...) — substitui o conteúdo do DIV_WINDOW sem recarregar a página inteira
Acesso direto pelo browser (URL)getOutput().write(this, page) — escreve o HTML completo com CSS, JS e título da janela

window() monta a estrutura de layout em linhas. Table aqui é um componente de layout do framework — não é uma tabela HTML de dados:

LinhaConteúdo
"99%" + DIV_WSPACEA ListView ocupa quase toda a altura. DIV_WSPACE envolve a lista para scroll interno
"1%" + line()Separador horizontal entre a lista e a barra inferior
"1%" + DIV_BOTTOMBarra inferior com navegação e botão "Novo Registro"
"1%" + line()Separador na base
Capítulo 4 — br() ▼ 
public Bar br() throws Exception {
    Button cmd_novo = ui().button("&nbsp;&nbsp;Novo Registro&nbsp;&nbsp;")
            .setCss("btn btn-success btn-lg")
            .setNoSize()
            .setOnClick(link(ModuloProdutoForm.ShowInsert.class)
                    .modal(new ModalConfig()
                            .setWidth("750")
                            .setHeight("570")
                            .setOnCloseURL(url(Sort.class))));

    return ui().bar()
            .addCenter(lView().nav(lView().getNavForm()))
            .addRight(cmd_novo);
}
PosiçãoComponente
CentrolView().nav(lView().getNavForm()) — navegação de páginas: anterior, próxima, total de registros
DireitaBotão "Novo Registro" — abre o formulário em modal 750×570

setOnCloseURL(url(Sort.class)) — ao fechar o modal (depois de salvar ou cancelar), o framework executa automaticamente a action Sort para recarregar a lista. Sem isso, o novo registro só apareceria após recarregar a página manualmente.

Capítulo 5 — lView() — a ListView ▼ 
private ListView lv = null;
public ListView lView() throws Exception {
    if (lv == null) {
        lv = ui().lView();

        String orderBy = getSession().getString(LIST.concat(lv.getORDER_BY()), getInput());
        if (orderBy == null) orderBy = ModuloProdutoBean.NOME_PRO;

        lv.setSortAct(url(Sort.class))
          .setPageAction(url(Sort.class))
          .setPage(getSession().getInteger(LIST.concat(lv.getPAGE()), getInput()))
          .setOrderBy(orderBy)
          .setSort(getSession().getString(LIST.concat(lv.getSORT()), getInput()))
          .ajax();

        lv.setPageSize(50);

        lv.setFiltro(proWBean());
        getFactory().modulo().proModel().daoList(lv.getData());

        ListColumn col_nome = lv.newColumn("Nome")      .setOrderBy(ModuloProdutoBean.NOME_PRO).setWidth(220).setPadding(";padding:10 8 10 8;");
        ListColumn col_vl   = lv.newColumn("Valor")     .setOrderBy(ModuloProdutoBean.VL_PRO)  .setWidth(100).setPadding(";padding:10 8 10 8;").alignCenter();
        ListColumn col_qtd  = lv.newColumn("Qtd")       .setOrderBy(ModuloProdutoBean.QTD_PRO) .setWidth(80) .setPadding(";padding:10 8 10 8;").alignCenter();
        ListColumn col_obs  = lv.newColumn("Observação").setOrderBy(ModuloProdutoBean.OBS_PRO)              .setPadding(";padding:10 8 10 8;");
        ListColumn col_del  = lv.newColumn("")                                                  .setWidth(40) .setPadding(";padding:6 4 6 4;") .alignCenter();

        while (lv.hasNext()) {
            ModuloProdutoBean bean = (ModuloProdutoBean) lv.next();

            ListLine line = lv.createLine()
                    .setOnclick(link(ModuloProdutoForm.ShowUpdate.class)
                            .putInteger(ModuloProdutoBean.ID_PRO, bean.getId_pro())
                            .modal(new ModalConfig()
                                    .setWidth("750")
                                    .setHeight("570")
                                    .setOnCloseURL(url(Sort.class))));

            col_nome.setContent(bean.getNome_pro());
            col_vl  .setContent(bean.getVl_pro());
            col_qtd .setContent(bean.getQtd_pro());
            col_obs .setContent(bean.getObs_pro());
            col_del.setHtmlData(
                "<button type='button' class='close' " +
                "style='opacity:0.6;font-size:20px;color:#d9534f;float:none;' " +
                "title='Excluir' " +
                "onclick=\"event.stopPropagation();" +
                link(DeleteFromList.class).putInteger(ModuloProdutoBean.ID_PRO, bean.getId_pro()).ajax() +
                "\">&times;</button>"
            );

            lv.addLine(line);
        }
    }
    return lv;
}

Inicialização lazy

Mesmo padrão do proBean() na Action: o componente só é criado na primeira chamada e reutilizado nas seguintes dentro da mesma requisição.

ORDER BY padrão

O campo de ordenação é lido da sessão. Se ainda não existe (primeira carga da tela), usa NOME_PRO como padrão — garante que a lista nunca aparece sem ordenação definida.

Configuração de paginação e ordenação

ChamadaO que configura
setSortAct(url(Sort.class))URL chamada ao clicar num cabeçalho de coluna
setPageAction(url(Sort.class))URL chamada ao navegar entre páginas
setPage(...)Página atual — lida da sessão ou do input
setOrderBy(orderBy)Coluna de ordenação atual
setSort(...)Direção: ASC ou DESC
ajax()Paginação e ordenação via Ajax — sem recarregar a página toda

setFiltro e daoList

setFiltro(proWBean()) associa o WBean como Criteria — define as condições WHERE aplicadas pelo daoList. Mesmo sem filtros ativos neste CRUD simples, a estrutura já está pronta para quando filtros forem adicionados.

daoList(lv.getData()) executa a consulta no banco e preenche a JasapList interna da ListView, já com paginação e ordenação aplicadas.

Colunas

ColunaCampoDetalhe
NomeNOME_PROLargura 220, ordenável
ValorVL_PROLargura 100, centralizado, ordenável
QtdQTD_PROLargura 80, centralizado, ordenável
ObservaçãoOBS_PROLargura livre, ordenável
(exclusão)—Largura 40, sem cabeçalho — botão × inline

Loop e linhas clicáveis

while (lv.hasNext()) itera sobre os registros da página atual. Para cada um, lv.createLine().setOnclick(...) cria uma linha clicável que abre o formulário de edição em modal — passando o ID do produto e configurando setOnCloseURL para recarregar a lista ao fechar.

setContent vs setHtmlData

MétodoComportamentoUsar quando
setContent(valor)Escapa HTML automaticamente — texto simplesDados do bean (nome, valor, obs)
setHtmlData(html)HTML bruto — não escapaBotões, ícones, qualquer HTML na célula

Botão de exclusão inline

A coluna col_del recebe um botão Bootstrap class='close' (×) com duas características importantes:

  • float:none — neutraliza o float:right padrão da classe close, necessário para que alignCenter() funcione corretamente
  • event.stopPropagation() — impede que o clique no botão acione o setOnclick da linha (que abriria o formulário de edição)
Capítulo 6 — DeleteFromList ▼ 
public static class DeleteFromList extends ModuloProdutoList {
    @Override
    public Effect execute() throws Exception {
        try {
            Integer id = getInput().getInteger(ModuloProdutoBean.ID_PRO);
            if (JasapFunctions.equals(getInput().getInteger(CONFIRM_LIST), 1)) {
                proBean().setId_pro(id);
                getFactory().modulo().proModel().daoDelete(proBean());
                update(lView().getDIV_BODY(),     lView().getBody());
                update(lView().getDIV_NAVIGATE(), lView().getNavForm());
            } else {
                eval(Js.swalConfirm(
                        "Confirma a exclusão desse registro?",
                        link(DeleteFromList.class)
                                .putInteger(ModuloProdutoBean.ID_PRO, id)
                                .putInteger(CONFIRM_LIST, 1)
                                .ajax(),
                        ""));
            }
            return new Response();
        } catch (SQLConstraintException e) {
            eval(new Toast("Falha ao excluir: esse registro está sendo usado em outra tabela.").fail());
            return new Response();
        }
    }
}

A exclusão usa um padrão de confirmação em duas etapas para evitar exclusões acidentais:

ChamadaO que acontece
1ª (CONFIRM_LIST ausente)Exibe o diálogo swalConfirm. O link do confirm repassa o ID e CONFIRM_LIST=1
2ª (CONFIRM_LIST == 1)Executa daoDelete e atualiza o corpo e a navegação da lista via Ajax

Por que o ID vem do getInput() e não do id_pro()? Porque ShowUpdate não foi chamado — o usuário clicou no × direto na lista, sem abrir o formulário. O ID não está na sessão; ele vem do parâmetro passado no link do botão.

SQLConstraintException é capturada para exibir um Toast vermelho quando o registro não pode ser excluído por estar referenciado em outra tabela (FK).
