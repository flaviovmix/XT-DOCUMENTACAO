ModuloProdutoWBean novo arquivo

O WBean (Where Bean) é o objeto de filtro da listagem. Ele estende o Bean e implementa a interface Criteria do Jasap — é quem carrega os parâmetros de busca que o DAO usa para montar o WHERE do SQL.

package br.xt.app.modulo.produto;

import br.jasap.dao.Criteria;

public class ModuloProdutoWBean extends ModuloProdutoBean implements Criteria {

    private String where;
    private String in_sql    = null;
    private String notin_sql = null;

    public String getIn_sql()                    { return in_sql; }
    public void   setIn_sql(String in_sql)       { this.in_sql = in_sql; }

    public String getNotin_sql()                 { return notin_sql; }
    public void   setNotin_sql(String notin_sql) { this.notin_sql = notin_sql; }

    public String getWhere()             { return where; }
    public void   setWhere(String where) { this.where = where; }

}

public class ModuloProdutoWBean extends ModuloProdutoBean implements Criteria

ElementoO que significa
publicVisível para o DAO e para a listagem, que instanciam o WBean diretamente
ModuloProdutoWBeanConvenção do projeto: mesmo prefixo do Bean + WBean
extends ModuloProdutoBeanHerda as constantes de campo do Bean (NOME_PRO, ID_PRO, etc.), que o daoWhere() usa ao montar o SQL. É também a convenção do projeto — todo WBean estende seu Bean correspondente
implements CriteriaContrato exigido pelo Jasap para que o framework reconheça o objeto como portador de filtros SQL. Sem isso, o daoWhere() não reconhece o objeto corretamente

private String where / in_sql / notin_sql

Os três campos são exigidos pela interface Criteria. O framework os lê ao montar o SQL final da consulta.

CampoO que armazena
whereA cláusula WHERE montada pelo daoWhere(). O DAO chama filtro.setWhere(where) ao final do método e o framework injeta esse valor no SQL antes de executar a consulta
in_sqlCláusula IN para filtros de múltipla seleção. Inicializado como null — não utilizado neste CRUD simples, mas exigido pela interface
notin_sqlCláusula NOT IN para exclusões de registros específicos. Inicializado como null — não utilizado neste CRUD simples, mas exigido pela interface

getWhere() / getIn_sql() / getNotin_sql()

Os três pares de getter/setter são exigidos pela interface Criteria — sem eles a classe não compila.

MétodoQuem chamaPara quê
getWhere()FrameworkLê a cláusula WHERE e injeta no <where> do SQL antes de executar
setWhere()daoWhere()Registra o resultado da montagem do WHERE ao final do método
getIn_sql()FrameworkLê a cláusula IN ao montar o SQL final
getNotin_sql()FrameworkLê a cláusula NOT IN ao montar o SQL final
