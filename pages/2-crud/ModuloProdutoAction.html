ModuloProdutoAction novo arquivo

Classe base abstrata de todas as actions de Produto. Centraliza três coisas que ModuloProdutoList, ModuloProdutoForm e suas inner classes precisam compartilhar: o prefixo ROOT, os objetos Bean e WBean da requisição, e o acesso ao ID do produto em edição.

package br.xt.app.modulo.produto;

import br.xt.AppsRootAction;
import br.xt.app.modulo.ModuloManager;

public abstract class ModuloProdutoAction extends AppsRootAction {

    public static final String ROOT = ModuloManager.F_ACESSO_MODULO.concat("__PRO/");

    private ModuloProdutoWBean wbean = null;
    public ModuloProdutoWBean proWBean() throws Exception {
        if (wbean == null)
            wbean = (ModuloProdutoWBean) getManager().getInstance(ModuloProdutoWBean.class);
        return wbean;
    }

    private ModuloProdutoBean bean = null;
    public ModuloProdutoBean proBean() throws Exception {
        if (bean == null)
            bean = (ModuloProdutoBean) getManager().getInstance(ModuloProdutoBean.class);
        return bean;
    }

    public Integer id_pro() throws Exception {
        return getSession().getInteger(ROOT.concat(ModuloProdutoBean.ID_PRO));
    }

}

public abstract class ModuloProdutoAction extends AppsRootAction

ElementoO que significa
abstractEsta classe nunca é usada diretamente — não tem execute(). Só existe para ser estendida por ModuloProdutoList e ModuloProdutoForm. O abstract impede que o framework tente instanciá-la ou registrá-la por engano
extends AppsRootActionHerda toda a infraestrutura do framework: getInput(), getSession(), getManager(), ui(), link(), eval(), update(), entre outros

public static final String ROOT = ModuloManager.F_ACESSO_MODULO.concat("__PRO/")

Prefixo único da entidade Produto. Usado para montar chaves de sessão e constantes de formulário, garantindo que não haja conflito com outras entidades do módulo.

ElementoO que significa
public static finalAcessível de qualquer lugar, pertence à classe e não pode ser alterado — é uma constante verdadeira
F_ACESSO_MODULO.concat("__PRO/")Resulta em "XT...MODULO__PRO/". O sufixo __PRO/ identifica a entidade Produto. Se o módulo tiver outras entidades (ex: Pessoa), cada uma terá seu próprio sufixo (__PES/), evitando que chaves de sessão de entidades diferentes se sobreponham

proBean() e proWBean() — qual a diferença?

Os dois parecem iguais porque ModuloProdutoWBean estende ModuloProdutoBean — mas servem para propósitos completamente diferentes:

MétodoTipo retornadoUsado porPara quê
proBean()ModuloProdutoBeanFormulárioCarrega os dados do registro sendo criado ou editado. O Form preenche os campos a partir dele e grava de volta nele antes de salvar
proWBean()ModuloProdutoWBeanListagemCarrega os parâmetros de filtro e o estado de paginação. A List guarda na sessão entre requisições para lembrar a página atual

Os dois ficam aqui porque tanto ModuloProdutoList quanto ModuloProdutoForm estendem esta classe — ambos precisam de acesso fácil aos dois objetos sem redeclarar.

private ... = null / if (wbean == null) / getManager().getInstance()

O campo privado começa como null e só é instanciado na primeira chamada — padrão chamado de inicialização lazy. A partir daí o mesmo objeto é reutilizado em todas as chamadas seguintes dentro da mesma requisição.

LinhaO que faz
private ModuloProdutoBean bean = nullCampo privado inicializado como null — ainda não foi criado
if (bean == null)Só instancia na primeira chamada. Se já foi criado, pula direto para o return
getManager().getInstance(...)Retorna a instância registrada no escopo da requisição. Garante que todas as actions que chamam proBean() na mesma requisição recebem o mesmo objeto — se uma action popula o bean com dados do input, a próxima já encontra esses dados sem precisar ler de novo

id_pro()

Lê da sessão o ID do produto que está sendo editado no momento.

Quando o usuário clica numa linha da lista, o ShowUpdate grava o ID na sessão:

getSession().addInt(ROOT.concat(ModuloProdutoBean.ID_PRO), proBean().getId_pro());

A partir daí, qualquer action que precisar saber qual produto está aberto chama id_pro() — o ID já está garantido na sessão, sem precisar ler do input novamente.

A chave ROOT.concat(ModuloProdutoBean.ID_PRO) resulta em "XT...MODULO__PRO/id_pro". O prefixo ROOT garante que essa chave não conflita com o ID de outra entidade que também tenha um campo id_xxx na sessão.
