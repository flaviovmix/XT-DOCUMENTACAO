ModuloProdutoDAO novo arquivo

O DAO é a única camada do módulo que fala diretamente com o banco. Todos os SQLs da entidade Produto passam por ele: listagem, busca por ID, insert, update e delete.

package br.xt.app.modulo.produto;

import br.jasap.core.AppManager;
import br.jasap.dao.Query;
import br.jasap.util.JasapList;
import br.xt.AppsRootDAO;

public class ModuloProdutoDAO extends AppsRootDAO {

    public ModuloProdutoDAO() {
    }

    public ModuloProdutoDAO(AppManager manager) {
        setManager(manager);
        setDataBase(manager.getDataBase());
    }

    public void daoList(JasapList list) throws Exception {
        StringBuilder sql = new StringBuilder();
        sql.append("select * from " + ModuloProdutoBean.TABLE);
        sql.append(" where 1=1 ");

        Query query = getDataBase().getQuery(sql.toString());
        select().executeList(query, list);

        while (query.next()) {
            ModuloProdutoBean bean = new ModuloProdutoBean();
            query.populateBean(bean);
            list.getList().add(bean);
        }
        query.release();
    }

    public void daoSingle(ModuloProdutoBean bean) throws Exception {
        select().execute(bean, ModuloProdutoBean.TABLE);
    }

    public void daoInsert(ModuloProdutoBean bean) throws Exception {
        insert().execute(bean, ModuloProdutoBean.TABLE);
    }

    public void daoUpdate(ModuloProdutoBean bean) throws Exception {
        update().execute(bean, ModuloProdutoBean.TABLE);
    }

    public void daoDelete(ModuloProdutoBean bean) throws Exception {
        delete().execute(bean, ModuloProdutoBean.TABLE);
    }

}

Por que daoList é diferente dos outros métodos?

Os métodos daoSingle, daoInsert, daoUpdate e daoDelete usam os helpers do Jasap — select().execute(), insert().execute(), etc. — que constroem o SQL automaticamente a partir das anotações do Bean (@DBInfo, pk=true, serial=true), executam e mapeiam o resultado, tudo internamente. Eles sempre operam em um único registro identificado pelo ID.

O daoList precisa fazer na mão porque retorna múltiplos registros, precisa de SQL customizado com <orderby> para ordenação, e precisa de paginação. Por isso monta o SQL, itera os resultados, cria um bean por linha e popula cada um.

daoList(JasapList list)

LinhaO que faz
sql.append("select * from " + ModuloProdutoBean.TABLE)Monta o SELECT. Usa a constante TABLE em vez de string literal — se a tabela mudar no banco, só atualiza o Bean
sql.append(" where 1=1 <orderby>")1=1 garante SQL válido mesmo sem condições. <orderby> é substituído pelo ORDER BY definido na listagem
getDataBase().getQuery(sql.toString())Prepara a query com o SQL montado — ainda não executou
select().executeList(query, list)Injeta <orderby>, aplica LIMIT/OFFSET para a página atual e executa a consulta no banco
while (query.next())Itera os registros retornados. next() retorna false quando não há mais linhas
query.populateBean(bean)Usa reflection para mapear cada coluna do ResultSet para o campo correspondente do bean — chama os setters pelo nome da coluna
list.getList().add(bean)Adiciona o bean populado na lista de resultados
query.release()Libera o cursor e devolve a conexão para o pool. Obrigatório — sem release() a conexão fica ocupada e o pool se esgota

daoSingle(ModuloProdutoBean bean)

Carrega um único registro pelo ID. O bean deve ter id_pro preenchido antes de chamar — o Jasap usa o campo marcado com @DBInfo(pk=true) para montar o WHERE automaticamente. Após a execução, os demais campos do bean são preenchidos com os dados do banco.

daoInsert(ModuloProdutoBean bean)

Insere um novo registro. O Jasap omite automaticamente o campo serial=true no INSERT. Após a execução, o ID gerado pelo banco é gravado de volta em bean.setId_pro().

daoUpdate(ModuloProdutoBean bean)

Atualiza um registro existente. O bean deve ter id_pro preenchido para o WHERE e os demais campos com os novos valores. O Jasap monta: UPDATE laboratorio.pro SET campo=valor, ... WHERE id_pro = ?

daoDelete(ModuloProdutoBean bean)

Exclui um registro pelo ID. Se houver FK em outra tabela apontando para esse registro, o banco lança SQLConstraintException — o Form captura essa exceção para exibir a mensagem de erro ao usuário.
